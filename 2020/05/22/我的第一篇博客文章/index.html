我的第一篇博客文章

Hello World——从0开始搭建个人博客（Hexo）

一、下载Node.js(框架的基础环境)

此步将安装node.js本身和包管理器（npm）

  1. 查看node.js及npm版本(cnpm及hexo查看版本命令类似)

    node -v

    npm -v

  2. 切换cnpm镜像源，确保npm安装的稳定及快速

    npm install -g cnpm --registry=https://registry.npm.taobao.org

  3. 安装hexo框架

    cpnm install -g hexo-cli

二、创建blog（win平台）

  1. 在E盘下创建blog文件夹

    cd E:

    mkdir blog

    查看当前路径

    echo %cd%

  2. 初始化博客

    hexo init

    windows平台需要先安装git环境，使用git的cmd软件，切换到blog路径，执行hexo初始化命令

  3. 启动hexo

    hexo s(server)

  4. 访问博客

    localhost:4000

三、新建一篇自己的博客

  1. 继上述操作后，回到cmd，执行 crtl + c选择断开操作

  2. 新建博客

    hexo n "我的第一篇博客文章"

  3. 可使用文本编辑器打开source\_posts路径下的博客文章

  4. 退回到blog目录下，重新生成

    hexo clean

    hexo g(generate)

四、部署个人博客到github

  1. 在github中新建repository，命名为github用户名.github.io，必须符合此要求！(此处失败两次，网页状态码500，第三次建仓库，在个人描述中改成了我的hexo博客后成功，神奇？)

  2. 安装git部署插件

    cnpm install --save hexo-deployer-git

  3. 修改blog路径下_config.yml配置（win平台可用记事本打开）

    冒号后要加空格！

  4. 部署到github

    hexo d(deploy)

    此处报错： Deployer not found：git

    原因：缺少了对本地电脑的github用户的设置，此用户的作用是告诉github远程仓库，部署者是谁。

    1. 打开gitbash，输入如下命令：

      git init

      git config --global user.name "name"

      git config --global user.email "email"

    2. 仍然不能部署，此时重新安装git部署插件后成功

      cnpm install --save hexo-deployer-git

五、更换主题

1.clone主题，依旧是blog路径下

git clone 主题url themes 存放路径

2.修改_config.yml

theme字段中修改相应的主题所在文件夹名称
